Xybrid 注册表中的每个模型都完全在设备端运行 — 无需云账号或 API Key。模型只需下载一次，会缓存到本地，并自动解析为最适合你平台的变体。

<Callout type="info">
注册表是唯一权威来源。下面的表格只是一份快照 — 运行 `xybrid models list`（或 `xybrid models search <query>`）查看实时目录。
</Callout>

## 模型命名

模型 ID 遵循 `{family}-{variant}-{version}` 模式：

- `kitten-tts-nano-0.2` — 家族、变体、上游模型版本
- `kokoro-82m` — 只有一个版本时不带版本后缀
- 版本指的是**上游模型版本**，不是 Xybrid 的集成版本

注册表条目一经发布即不可变 — 新的上游版本会成为新的注册表条目。

## 使用模型

```bash
# CLI
xybrid models info kokoro-82m
xybrid run --model kokoro-82m --input-text "Hello!" -o hello.wav
```

```dart
// Flutter — 所有 SDK 使用相同的模型 ID
final model = await Xybrid.model('kokoro-82m').load();
```

下载内容缓存在 `~/.xybrid/cache/` — 参见[模型缓存](/docs/concepts/caching)。

## 注册表之外的模型

你也可以运行不在注册表中的模型：

- **HuggingFace GGUF**：`xybrid run --huggingface <repo>[:variant]`（或 SDK 中的 `fromHuggingFace`）直接下载 GGUF
- **本地 Bundle**：加载 `.xyb` Bundle 或模型目录
- **自定义模型**：编写自己的 `model_metadata.json` — 参见[自定义模型指南](https://github.com/xybrid-ai/xybrid/blob/master/docs/guides/custom-models.zh.md)
