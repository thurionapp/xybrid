# Xybrid Expo example

A minimal Expo app that exercises `react-native-xybrid` end-to-end:
initialize → load a model from the registry → run one inference, with
per-step latency. Use it to smoke-test the binding on a real device or
simulator.

## ⚠️ Development build required — NOT Expo Go

`react-native-xybrid` ships custom native code (a TurboModule + a prebuilt
Rust/UniFFI framework). **Expo Go cannot load it.** You must run a
[development build](https://docs.expo.dev/develop/development-builds/introduction/)
via `expo run:ios` / `expo run:android` (or EAS). The native `ios/` and
`android/` folders are generated by `expo prebuild` (Continuous Native
Generation) and are gitignored — they are not committed.

## Prerequisites

- The binding's **native artifacts must be staged first**. They are produced
  from the Rust core and are not checked in:

  ```bash
  # from the repo root
  cargo xtask build-react-native            # both platforms
  # or, for a single platform:
  cargo xtask build-react-native --skip-android   # iOS only
  cargo xtask build-react-native --skip-ios       # Android only
  ```

  This must run **before** `expo prebuild` / `expo run:*`, or autolinking
  won't find the XCFramework / `.so` files.

  > iOS note: a real iOS slice requires ONNX Runtime for iOS vendored at
  > `vendor/ort-ios/`. Without it, `build-xcframework` falls back to a
  > macOS-only framework and `build-react-native` refuses to stage it unless
  > you pass `--allow-macos-only` (which produces an iOS-incompatible package).

- iOS: Xcode + an iOS Simulator. Android: Android SDK + NDK + an emulator
  (AVD) or a connected device.

## Run

```bash
cd bindings/react-native/example
npm install            # symlinks react-native-xybrid from ../

# iOS (macOS only)
npm run ios            # = expo run:ios — prebuild + pod install + xcodebuild

# Android
npm run android        # = expo run:android — prebuild + gradle + install
```

Tap **Run smoke test**. Each step (initialize, load, run) is timed; failures
surface the underlying Xybrid error code. The default model is `whisper-tiny`
(editable in the field) — try `kokoro-82m` to exercise the TTS voices path.

## Notes

- **New Architecture** is required and is the Expo SDK 56 default — no extra
  config needed.
- After changing anything in the parent package's native layer, re-stage the
  artifacts and re-run `npm run prebuild` (`expo prebuild --clean`) so the
  generated projects pick up the change.
- This is intentionally a single-screen smoke test, not a full demo app.
